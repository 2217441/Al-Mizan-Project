# Product Manager Runbooks

## 1. Discovery Phase
**Goal:** Understand users and market opportunity.

### Workflow
1. **User Research:** Interview users, analyze surveys, map journeys.
2. **Market Analysis:** Study competitors, identify trends.
3. **Problem Validation:** Confirm the core problems to solve.
4. **Solution Ideation:** Brainstorm and prototype solutions.
5. **Business Case:** Asses value, cost, and risk.

### Checklist
- [ ] User research conducted
- [ ] Market analysis completed
- [ ] Problem validated
- [ ] Solution ideation done
- [ ] Business case prepared
- [ ] Technical feasibility checked
- [ ] Resource assessment done
- [ ] Risk evaluation completed

## 2. Implementation Phase
**Goal:** Build and launch successful products.

### Workflow
1. **Requirements:** Define detailed requirements (user stories, specs).
2. **Prioritization:** Rank features using RICE or Value/Effort.
3. **Coordination:** Work with Engineering, Design, QA.
4. **Monitoring:** Track development progress (velocity, burndown).
5. **Feedback & Iteration:** Review builds, gather internal/beta feedback.
6. **Launch:** Execute GTM plan (marketing, sales enablement).

### Progress Tracking Template
```json

  "agent": "product-manager",
  "status": "building",
  "progress": {
    "features_shipped": 0,

```

## 3. Product Excellence Phase
**Goal:** Deliver products that drive growth and continuously improve.

### Workflow
1. **Measurement:** Track KPIs (Activations, Retention, Revenue).
2. **Analysis:** Deep dive into analytics (Funnel, Cohort).
3. **Optimization:** Run A/B tests, optimize UX.
4. **Growth:** Implement growth loops (Virality, SEO).
5. **Vision Check:** Ensure alignment with long-term strategy.

### Delivery Notification Template
"Product launch completed. Shipped [X] features achieving [Y]% user satisfaction and [Z]% adoption rate. Revenue impact [Amount] with [Growth]x user growth. NPS improved from [A] to [B]. Product-market fit validated with [C]% retention."
