# OpenSpec: Proposal Workflow

Use this workflow when you are about to start a new feature or complex refactor.

## Instructions
1.  **Analyze requirements**: Review the user's request and project context.
2.  **Create a design doc**: Draft a new file in `docs/proposals/` using the OpenSpec template.
3.  **Identify risks**: Security, performance, and complexity assessments.
4.  **Request review**: Use `notify_user` to present the proposal to the user.

## Checklist
- [ ] Problem defined
- [ ] Technical design outlined
- [ ] Security implications checked
- [ ] Verification plan included
